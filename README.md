# 📚 Online Bookstore Database & Sales Analytics: From Inventory to Insights

## 📌 Project Overview

This project focuses on designing and analyzing a relational database for an online bookstore.
Using SQL, the database was created and analyzed to extract business insights related to sales performance, customer purchasing behavior, and inventory management.

The goal of this project is to demonstrate practical SQL skills used by data analysts to transform raw transactional data into meaningful insights.

---

## 🗄️ Database Schema

The database consists of multiple relational tables connected through primary and foreign keys.

**Main Tables**

* **Books** – Contains information about book titles, authors, genre, and price.
* **Customers** – Stores customer details.
* **Orders** – Records purchase transactions.
* **Order_Details** – Contains order items and quantities.

These tables form a structured relational database that allows efficient querying and analysis.

## 📈 Example SQL Query

```sql
-- Top Selling Books
SELECT 
    b.title,
    SUM(od.quantity) AS total_sales
FROM order_details od
JOIN books b ON od.book_id = b.book_id
GROUP BY b.title
ORDER BY total_sales DESC;
```

This query identifies the best-selling books by calculating the total number of units sold.
